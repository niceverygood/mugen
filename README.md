# MUGEN - 구조도면 자동생성 시스템

건축 DXF 도면을 업로드하면 구조 도면(기초, 토대, 스터드, 바닥, 천정, 지붕 등)을 자동 생성하는 웹 애플리케이션입니다.

## 1. 시스템 요구사항

| 항목 | 요구사항 |
|------|----------|
| Node.js | v22 이상 |
| pnpm | v10 이상 |
| 브라우저 | Chrome, Edge, Firefox (최신 버전) |
| Docker (선택) | Docker Compose v3.8 이상 |

## 2. 설치 및 실행

### 로컬 개발 환경

```bash
# 1. 의존성 설치
pnpm install

# 2. 데이터베이스 마이그레이션
pnpm db:migrate

# 3. 시드 데이터 생성 (초기 계정 및 프리셋)
pnpm db:seed

# 4. 개발 서버 실행 (클라이언트 :3000 / 서버 :3001)
pnpm dev
```

실행 후 브라우저에서 `http://localhost:3000` 으로 접속합니다.

### Docker 환경

```bash
docker-compose up --build
```

실행 후 `http://localhost` (포트 80)으로 접속합니다.

## 4. 사용 가이드

### 4.1 로그인

1. 브라우저에서 시스템에 접속하면 **로그인 화면**이 나타납니다.
2. 아이디와 비밀번호를 입력하고 **ログイン** 버튼을 클릭합니다.
3. 로그인에 성공하면 대시보드로 이동합니다.
4. 세션은 **8시간** 유지되며, 리프레시 토큰으로 **7일**까지 자동 연장됩니다.

### 4.2 대시보드 (프로젝트 관리)

로그인 후 진입하는 메인 화면입니다.

#### 프로젝트 목록 보기

- 모든 프로젝트가 카드 형태로 표시됩니다.
- 각 카드에는 프로젝트명, 상태 배지, 거래처명, 도면 수, 작성자, 최종 수정일이 표시됩니다.
- **상태 필터 버튼**으로 원하는 상태만 볼 수 있습니다:
  - `全て` (전체) / `作業中` (작업중) / `レビュー` (검토중) / `完了` (완료)

#### 새 프로젝트 만들기

1. 우측 상단의 **+ 新規プロジェクト** 버튼을 클릭합니다.
2. **프로젝트명**과 **거래처명**을 입력합니다.
3. **作成** 버튼을 클릭하면 프로젝트가 생성되고 프로젝트 상세 화면으로 이동합니다.

#### 프로젝트 열기

- 프로젝트 카드를 클릭하면 해당 프로젝트 상세 화면으로 이동합니다.

#### 로그아웃

- 우측 상단의 **ログアウト** 버튼을 클릭합니다.

### 4.3 프로젝트 상세

선택한 프로젝트의 정보와 도면 목록을 관리하는 화면입니다.

#### 프로젝트 정보

화면 상단에 거래처명, 층수, 지붕 타입, 프리셋 정보가 표시됩니다.

#### DXF 파일 업로드

1. 화면 중앙의 **DXFファイルをアップロード** 영역을 클릭합니다.
2. 건축 도면 파일(`.dxf`)을 선택합니다.
3. 업로드가 완료되면 도면 목록에 추가됩니다.

> 지원 형식: `.dxf` 파일만 업로드 가능 (최대 50MB)
> 일본 CAD 소프트웨어의 Shift-JIS 인코딩도 자동 지원합니다.

#### 도면 목록

업로드된 도면이 리스트로 표시됩니다. 각 도면에는 다음 정보가 포함됩니다:

| 항목 | 설명 |
|------|------|
| 타입 배지 | `意匠図面` (건축도면) / `構造図面` (구조도면) / `手動作成` (수동생성) |
| 파일명 | 업로드된 DXF 파일명 |
| 버전 | 도면 버전 (v1, v2, ...) |
| 생성일시 | 업로드 일시 |

#### 도면 관련 버튼

- **エディタで開く** - 건축 도면(`意匠図面`)을 DXF 에디터에서 엽니다
- **ダウンロード** - DXF 파일을 다운로드합니다

### 4.4 DXF 에디터 (핵심 기능)

구조 도면 자동 생성의 핵심 화면입니다. 3개 영역으로 구성됩니다.

```
┌──────────────────────────────────────────────────────┐
│  ← MUGEN  構造図面自動生成  ✋移動  ▬壁体    DXF開く │  ← 헤더
├──────┬───────────────────────────────────┬────────────┤
│ 좌측  │                                  │   우측      │
│ 패널  │         DXF 캔버스 (중앙)          │   패널      │
│      │                                   │            │
│ 4탭  │    건축도면 + 구조도면 오버레이       │  범례/통계  │
│      │                                   │            │
├──────┴───────────────────────────────────┴────────────┤
│           좌표 표시              줌 컨트롤 (+ − ⌂)     │
└──────────────────────────────────────────────────────┘
```

#### 좌측 패널 - 4개 탭

**1) 自動生成 (자동생성) 탭**

구조 도면을 자동으로 생성하는 핵심 기능입니다.

1. **프리셋 선택** - 거래처별 프리셋 중 하나를 클릭합니다.
   - 프리셋에는 벽체 타입(LGS/Wood/RC)과 스터드 간격이 표시됩니다.
   - 프리셋을 선택하지 않으면 자동생성 버튼이 비활성화됩니다.
2. **층수 선택** - `1階` 또는 `2階` 를 선택합니다.
3. **지붕 타입 선택** - `切妻` (박공지붕) 또는 `寄棟` (모임지붕)을 선택합니다.
4. **構造図面自動生成 버튼 클릭** - 서버에서 구조 도면 생성이 시작됩니다.
   - 생성 중에는 "生成中..." 으로 표시되며, 완료될 때까지 자동으로 폴링합니다.
5. **생성 완료 후** - 생성된 레이어 목록이 표시됩니다.
   - 각 레이어의 **표시/숨김**을 개별 토글할 수 있습니다.
   - `全表示` / `全非表示` 버튼으로 전체 레이어를 한번에 제어할 수 있습니다.
   - 레이어를 클릭하면 해당 레이어만 강조 표시됩니다.

**2) レイヤー (레이어) 탭**

업로드된 원본 DXF 파일의 레이어를 관리합니다.

- 각 레이어의 표시/숨김을 토글할 수 있습니다.
- 레이어를 클릭하면 해당 레이어만 강조(나머지 흐리게) 표시됩니다.
- 레이어 수와 엔티티 수가 표시됩니다.

**3) プリセット (프리셋) 탭**

등록된 모든 프리셋을 조회합니다.

- 거래처명, 스터드 간격, 벽체 타입, 최대 스팬, 비고를 확인할 수 있습니다.
- 현재 선택된 프리셋이 강조 표시됩니다.

**4) エラー (에러) 탭**

구조 검증 결과를 표시합니다.

- 빨간색: 오류 (수정 필요)
- 주황색: 경고 (확인 권장)
- 수동 작성한 벽체 요소에 대한 검증 결과가 표시됩니다.

#### 중앙 - DXF 캔버스

도면을 시각적으로 표시하는 메인 영역입니다.

- **원본 건축 도면** - 기본 색상으로 표시 (구조 도면 생성 후 흐리게 처리)
- **생성된 구조 도면** - 레이어별 고유 색상으로 오버레이 표시
- **수동 벽체** - 초록색(유효) / 빨간색(오류)으로 표시

**캔버스 조작:**

| 조작 | 방법 |
|------|------|
| 이동 (팬) | ✋ 이동 도구 선택 후 드래그 |
| 확대/축소 | 마우스 휠 스크롤 |
| 줌 컨트롤 | 우하단 `+` `-` `⌂`(초기화) 버튼 |
| 좌표 확인 | 좌하단에 마우스 위치의 DXF 좌표 표시 |

**도구:**

| 도구 | 설명 |
|------|------|
| ✋ 移動 | 캔버스를 드래그하여 이동 |
| ▬ 壁体 | 클릭 2번으로 벽체 요소를 수동 작성. 우클릭으로 취소 |

#### 우측 패널

| 섹션 | 설명 |
|------|------|
| **図面凡例** (범례) | 생성된 구조 레이어의 색상 범례 |
| **生成統計** (통계) | 레이어별 엔티티 수 + 총 엔티티 수 |
| **作業フロー** (작업흐름) | 5단계 체크리스트 (진행상황 자동 표시) |
| **단축키 안내** | `⌘Z` 실행취소, `ESC` 취소, 우클릭 작도취소 |

#### 헤더 버튼

| 버튼 | 설명 |
|------|------|
| **←** | 프로젝트 상세 화면으로 돌아가기 |
| **DXF開く** | 로컬 DXF 파일을 열어 미리보기 |
| **DXFエクスポート** | 생성된 구조 도면을 DXF 파일로 내보내기 |

## 5. 작업 흐름 요약

```
① 로그인
   ↓
② 프로젝트 생성 (또는 기존 프로젝트 선택)
   ↓
③ 건축 DXF 도면 업로드
   ↓
④ 에디터에서 도면 열기
   ↓
⑤ 프리셋 선택 + 층수/지붕 타입 설정
   ↓
⑥ "構造図面自動生成" 실행
   ↓
⑦ 생성된 레이어 확인/토글
   ↓
⑧ 필요시 수동 벽체 추가/수정
   ↓
⑨ DXF 내보내기 → JW CAD 등에서 활용
```

## 6. 프리셋 설명

프리셋은 거래처별 건축 사양을 미리 저장한 설정입니다.

| 프리셋명 | 벽체 타입 | 스터드 간격 | 최대 스팬 | 비고 |
|----------|----------|------------|----------|------|
| 거래처 A 기본 | LGS (경량철골) | 455mm | 4,550mm | 軽量鉄骨造 |
| 거래처 B 기본 | Wood (목조) | 303mm | 5,460mm | 木造軸組 |
| 거래처 C 기본 | RC (철근콘크리트) | 910mm | 8,000mm | RC造 |

- **ADMIN** 역할만 프리셋을 추가/수정/삭제할 수 있습니다.
- 프로젝트에 프리셋을 지정하면 해당 사양으로 구조 도면이 생성됩니다.

## 10. Docker 배포

### 서비스 구성

| 서비스 | 설명 | 포트 |
|--------|------|------|
| `postgres` | PostgreSQL 16 데이터베이스 | 5432 |
| `server` | Express.js API 서버 | 3001 |
| `client` | React SPA (Nginx) | 3000 |
| `nginx` | 리버스 프록시 | 80 |

### 환경 변수

```bash
# .env 파일에 설정 (선택)
DB_PASSWORD=mugen_secure_2024        # DB 비밀번호
JWT_SECRET=your-jwt-secret           # JWT 시크릿
JWT_REFRESH_SECRET=your-refresh-secret  # JWT 리프레시 시크릿
```

### 실행

```bash
# 빌드 및 실행
docker-compose up --build -d

# 로그 확인
docker-compose logs -f

# 종료
docker-compose down
```

### 데이터 영속성

| 볼륨 | 경로 | 설명 |
|------|------|------|
| `postgres_data` | `/var/lib/postgresql/data` | 데이터베이스 |
| `upload_data` | `/data/uploads` | 업로드된 DXF 파일 |

## 11. API 레퍼런스

기본 URL: `http://localhost:3001/api`

### 인증

| 메서드 | 경로 | 설명 | 인증 |
|--------|------|------|------|
| `POST` | `/auth/login` | 로그인 | 불필요 |
| `POST` | `/auth/refresh` | 토큰 갱신 | 불필요 |
| `GET` | `/auth/me` | 현재 사용자 조회 | 필요 |

**로그인 요청:**
```json
{
  "username": "admin",
  "password": "Mugen2024!"
}
```

**로그인 응답:**
```json
{
```

## 12. 문제 해결

### 로그인이 안 됩니다

- 시드 데이터가 생성되었는지 확인: `pnpm db:seed`
- 아이디/비밀번호 확인: `admin` / `Mugen2024!`

### DXF 파일 업로드가 실패합니다

- `.dxf` 확장자 파일만 지원됩니다.
- 파일 크기가 50MB를 초과하지 않는지 확인하세요.
- 서버의 업로드 디렉토리 (`uploads/` 또는 `/data/uploads/`)에 쓰기 권한이 있는지 확인하세요.

### DXF 파일이 깨져서 표시됩니다

- 일본 CAD 소프트웨어(JW CAD 등)로 생성된 Shift-JIS 인코딩 파일은 자동 감지됩니다.
- 그래도 깨지면 UTF-8로 변환 후 재업로드를 시도하세요.

### 자동 생성 버튼이 비활성화됩니다

- **프리셋이 선택되었는지** 확인하세요 (자동생성 탭에서 프리셋 클릭).
- DXF 도면이 정상적으로 로드되었는지 확인하세요.

### 생성 결과가 예상과 다릅니다

- 원본 건축 도면에 충분한 축선(구조축)이 포함되어 있는지 확인하세요.
- 축선이 부족할 경우 바운딩 박스 기반으로 자동 합성되므로 결과가 단순해질 수 있습니다.
- 프리셋의 스터드 간격, 최대 스팬 설정이 도면에 적합한지 확인하세요.

### Docker 실행 시 DB 연결 오류

- PostgreSQL 컨테이너가 정상 기동되었는지 확인: `docker-compose ps`
- 헬스체크가 통과될 때까지 서버가 대기하므로, 초기 기동 시 수십 초 소요될 수 있습니다.
